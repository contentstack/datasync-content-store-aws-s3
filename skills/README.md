# Skills – Contentstack DataSync Content Store (AWS S3)

Source of truth for detailed guidance. Read [AGENTS.md](../AGENTS.md) first, then open the skill that matches your task.

## When to use which skill

| Skill folder | Use when |
| --- | --- |
| [dev-workflow](dev-workflow/) | Setting up locally, running build/test/lint, Husky, or understanding CI workflows. |
| [content-store-aws-s3-sdk](content-store-aws-s3-sdk/) | Changing or documenting the public API, config shape, S3/content-store behavior, or integration with DataSync Manager. |
| [typescript](typescript/) | TypeScript or `tsconfig`/layout conventions in this repo. |
| [testing](testing/) | Writing or refactoring Jest tests, coverage, or test doubles (e.g. nock). |
| [code-review](code-review/) | Preparing or reviewing a PR for this repository. |
| [framework](framework/) | AWS SDK usage, env/credentials concerns, runtime/deployment assumptions. |

Each folder contains `SKILL.md` with YAML frontmatter (`name`, `description`).
